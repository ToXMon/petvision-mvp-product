# PetVision PDF Reports API Documentation

## Overview

The PetVision PDF Report Generation API provides endpoints for generating, streaming, and sharing professional health reports in PDF format. All reports include pet information, scan findings, AI analysis, and veterinary recommendations.

## Base URL

```
https://api.petvision.ai/api/reports
```

## Authentication

All API requests require authentication via JWT token in the Authorization header:

```
Authorization: Bearer <your-jwt-token>
```

## Endpoints

### 1. Generate PDF (Download)

Generate and download a PDF report as a file attachment.

**Endpoint:** `POST /api/reports/generate-pdf`

**Query Parameters (GET):**
- `scan_result_id` (string, required) - The ID of the scan result
- `report_type` (string, optional) - Report type: `summary` or `detailed` (default: `detailed`)
- `include_qr_code` (boolean, optional) - Include QR code linking to digital report (default: `true`)

**Request Body (POST):**
```json
{
  "scan_result_id": "scan-123",
  "report_type": "detailed",
  "include_qr_code": true
}
```

**Response:**
- **Content-Type:** `application/pdf`
- **Content-Disposition:** `attachment; filename="PetVision-Report-{reportId}.pdf"`
- **Headers:**
  - `X-Report-ID` - The generated report ID
  - `Content-Length` - File size in bytes

**Example:**
```bash
curl -X POST "https://api.petvision.ai/api/reports/generate-pdf" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "scan_result_id": "scan-123",
    "report_type": "detailed",
    "include_qr_code": true
  }' \
  --output report.pdf
```

### 2. Stream PDF (View in Browser)

Stream a PDF report directly to the browser for inline viewing.

**Endpoint:** `POST /api/reports/stream-pdf`

**Query Parameters (GET):**
- `scan_result_id` (string, required) - The ID of the scan result
- `report_type` (string, optional) - Report type: `summary` or `detailed` (default: `detailed`)
- `include_qr_code` (boolean, optional) - Include QR code (default: `true`)

**Request Body (POST):**
```json
{
  "scan_result_id": "scan-123",
  "report_type": "detailed",
  "include_qr_code": true
}
```

**Response:**
- **Content-Type:** `application/pdf`
- **Content-Disposition:** `inline`
- **Cache-Control:** `public, max-age=3600`

**Example:**
```bash
curl -X POST "https://api.petvision.ai/api/reports/stream-pdf" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"scan_result_id": "scan-123"}'
```

### 3. Share via Email

Email a PDF report to a recipient.

**Endpoint:** `POST /api/reports/share-email`

**Request Body:**
```json
{
  "scan_result_id": "scan-123",
  "email": "vet@example.com",
  "recipient_name": "Dr. Smith",
  "report_type": "detailed",
  "include_qr_code": true
}
```

**Response:**
```json
{
  "success": true,
  "message": "PDF report sent successfully",
  "reportId": "report-123",
  "email": "vet@example.com"
}
```

**Example:**
```bash
curl -X POST "https://api.petvision.ai/api/reports/share-email" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "scan_result_id": "scan-123",
    "email": "vet@example.com",
    "recipient_name": "Dr. Smith"
  }'
```

## Report Types

### Summary Report
- **Pages:** 1-2 pages
- **Content:**
  - Pet information
  - Scan summary
  - Key findings with severity
  - Main recommendations
- **Use Case:** Quick overview, sharing with pet owners

### Detailed Report
- **Pages:** 3-10 pages (varies by findings)
- **Content:**
  - Full cover page with pet photo
  - Complete findings table
  - Detailed analysis per finding
  - AI confidence scores
  - Comprehensive recommendations
  - Trend analysis (if available)
  - Comparison to previous scans
  - QR code for digital access
- **Use Case:** Veterinary consultation, medical records

## Response Codes

| Code | Description |
|------|-------------|
| 200  | Success - PDF generated or sent |
| 400  | Bad Request - Missing or invalid parameters |
| 401  | Unauthorized - Invalid or missing token |
| 404  | Not Found - Scan result does not exist |
| 500  | Internal Server Error - PDF generation failed |

## Error Response Format

```json
{
  "success": false,
  "error": "Error message describing the issue"
}
```

## Rate Limiting

- **Generate PDF:** 10 requests per minute per user
- **Stream PDF:** 10 requests per minute per user
- **Share Email:** 5 requests per minute per user

Rate limit headers are included in responses:
- `X-RateLimit-Limit` - Request limit
- `X-RateLimit-Remaining` - Remaining requests
- `X-RateLimit-Reset` - Reset timestamp

## Caching

- PDFs are cached for 1 hour (3600 seconds)
- Reports can be regenerated by adding `cache=false` parameter
- Cached responses include `X-Cache: HIT` header

## Best Practices

1. **Use summary reports** for quick shares with pet owners
2. **Use detailed reports** for veterinary consultations
3. **Include QR codes** for easy digital access
4. **Cache PDFs** to reduce server load
5. **Implement retry logic** for network failures
6. **Validate email addresses** before sending
7. **Use GET requests** for simple downloads
8. **Use POST requests** for complex configurations
